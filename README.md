# 报销信息收集系统

基于 Flask 的报销信息管理系统，包含移动端填报和 PC 端管理后台。

## 功能特点

### 移动端（前端）
- ✅ 授权码验证
- ✅ 报销信息填报（日期、部门、报销人、用途、金额）
- ✅ 附件上传（支持图片和 PDF）
- ✅ 安全保护（未授权访问自动跳转）

### PC 端（管理后台）
- ✅ 管理员和操作员登录
- ✅ 数据统计看板
- ✅ 报销信息管理（查看、搜索、筛选、分页）
- ✅ 审核流程（通过/驳回）
- ✅ 授权码管理（生成、启用/禁用，仅管理员）
- ✅ 用户管理（增删改密码，仅管理员）
- ✅ 操作日志记录（登录、审核等操作）

## 安装步骤

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 运行系统

```bash
python app.py
```

首次运行会自动创建数据库和默认管理员账号。

### 3. 访问系统

**移动端（填报页）**：http://localhost:5000/

**PC 端（管理后台）**：http://localhost:5000/admin/login

### 4. 默认管理员账号

- 用户名：`admin`
- 密码：`123456`

⚠️ **重要**：首次登录后请立即修改密码！密码长度至少为6位。

## 使用说明

### 用户角色权限

系统支持两种用户角色：

**管理员（admin）**：
- 查看数据统计
- 管理报销信息（查看、审核）
- 管理授权码（生成、启用/禁用）
- 管理用户（创建、删除、修改密码）
- 查看操作日志

**操作员（operator）**：
- 查看数据统计
- 管理报销信息（查看、审核）
- 查看操作日志

### 管理员操作流程

1. 登录管理后台（http://localhost:5000/admin/login）
2. 进入"授权码"页面，生成授权码
3. 将生成的授权码提供给报销人

### 报销人操作流程

1. 访问移动端首页（http://localhost:5000/）
2. 输入管理员提供的授权码
3. 验证通过后进入填报页
4. 填写报销信息：
   - 报销日期
   - 报销部门
   - 报销人姓名
   - 用途描述
   - 金额
   - 上传附件（可选，支持 JPG/PNG/PDF）
5. 提交成功

### 审核流程

1. 管理员登录后台
2. 在"报销管理"中查看待审核记录
3. 使用搜索和筛选功能快速定位需要审核的记录
4. 点击"详情"查看详细信息
5. 填写审核意见，选择"通过"或"驳回"

### 分页功能

- 报销管理页面采用分页显示，每页显示20条记录
- 分页导航保留当前的搜索和筛选条件
- 显示当前页的数据范围和总记录数
- 支持上一页、下一页和页码跳转

## 数据库说明

使用 SQLite 数据库，文件名为 `reimburse.db`，自动生成在项目根目录。

### 数据表结构

- **User**: 管理员用户表
- **User**: 管理员用户表
- **AuthCode**: 全局授权码表（唯一）
- **Reimbursement**: 报销信息表
- **Attachment**: 附件表
- **AuditLog**: 审核日志表
- **OperationLog**: 操作日志表

## 文件上传

上传的文件保存在 `uploads/` 目录下，文件名会自动添加 UUID 前缀以防重名。

## 配置说明

在 `app.py` 中可以修改以下配置：

- `SECRET_KEY`: Flask 密钥（生产环境务必修改）
- `SQLALCHEMY_DATABASE_URI`: 数据库连接
- `UPLOAD_FOLDER`: 文件上传目录
- `MAX_CONTENT_LENGTH`: 最大上传文件大小（默认 16MB）

在 `routes.py` 中可以修改以下配置：

- `per_page`: 每页显示的记录数（默认为20条）

## 安全建议

1. 修改默认管理员密码
2. 修改 SECRET_KEY 为随机字符串
3. 生产环境使用 HTTPS
4. 定期备份数据库
5. 限制上传文件类型和大小

## 技术栈

- **后端框架**: Flask 3.0
- **ORM**: SQLAlchemy
- **登录管理**: Flask-Login
- **数据库**: SQLite
- **前端**: HTML5 + CSS3（响应式设计）

## 清理脚本

项目提供了 `clean.py` 脚本，用于清理临时文件和数据库：

```bash
python clean.py
```

该脚本会清理以下内容：
- 所有 `__pycache__` 目录
- `uploads` 文件夹中的所有文件
- `reimburse.db` 数据库文件

⚠️ **注意**：运行清理脚本会删除所有数据，请谨慎使用！

## 常见问题

### Q: 忘记管理员密码怎么办？
A: 删除 `reimburse.db` 文件，重新运行程序会重置数据库和默认密码。

### Q: 如何创建操作员账号？
A: 管理员登录后，进入"用户"页面，点击"新建用户"，角色选择"操作员"即可。

### Q: 操作员可以做什么？
A: 操作员可以查看数据统计、管理报销信息（查看、审核）和查看操作日志，但不能管理用户和授权码。

### Q: 上传文件失败？
A: 检查 `uploads/` 目录是否有写权限，确认文件大小不超过限制。

### Q: 授权码无效？
A: 检查授权码是否被禁用或输入错误。

### Q: 操作日志为什么是空的？
A: 操作日志会在管理员登录、审核报销等操作时自动记录。如果日志为空，请先进行一些操作。
